# EDIE Runner ‚Äî Phase 2 (Art Pass) Implementation Plan

**Goal:** Replace every greybox rectangle with cohesive pixel art. EDIE comes from the user-provided reference images (downsampled + palette-quantized). All other assets (obstacles, background, Aurora Stones, UI font) are produced by a single deterministic Python generator script committed to the repo.

**Architecture:** A `tools/generate_art.py` script reads the user EDIE refs from `assets/source/`, processes them, and writes all game-ready PNGs to `assets/gen/`. A `scripts/copy_assets.{sh,ps1}` step copies `assets/gen/*.png` into `web/`. Rust loads textures via macroquad's `load_texture` in a new `assets::load_all()` async function. A new `Loading` game state runs while textures load. `render::sprites` swaps `draw_rectangle` bodies for `draw_texture_ex` while keeping function signatures stable.

**Reviewer-driven design decisions** (from Designer Persona Review):
- Author at native pixel res, render with `FilterMode::Nearest`, integer camera scale.
- Obstacles need 2px dark outline + cool-tone accents to pop off the warm background.
- 4-layer parallax (sky, far, mid, floor) with bg-far hue-shifted slightly cooler.
- EDIE shadow ellipse (anti-floating).
- Dash trail = 6+ ghost frames, alpha fade.
- Telegraph frames on QuadDrone/SparkBurst (200ms red blink before contact).
- Vertical slice gate triplet: **EDIE run + QuadDrone + bg_mid** (NOT cable + floor).
- Pixel font: `m5x7` (free, by Daniel Linssen).
- Palette lock: every output PNG must contain ‚â§16 unique colors.

**Out of scope (Phase 3):** particles, screen shake, score popups, slow-mo VFX, SFX, vignette, quad-storage fix.
